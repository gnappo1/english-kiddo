## Contributing

Thanks for your interest in helping.

This project is a small, evolving English-learning site for a child, so changes should stay:
- simple
- kid-friendly
- easy to maintain
- safe for a static website

## Ways to contribute

### Content
- Add vocabulary sets (small batches)
- Improve minimal pairs (common pronunciation traps)
- Add kid-friendly sentence prompts (very short)

### UX and UI
- Improve tablet layout
- Improve animations (keep them smooth and not distracting)
- Add accessibility improvements (keyboard support, ARIA labels)

### Code
- Fix bugs
- Refactor for clarity
- Add small utilities (no heavy frameworks)

## Pull request guidelines

1. Keep PRs small and focused
2. Avoid big rewrites
3. Prefer plain HTML/CSS/JS
4. If you add images, keep them local under `assets/` and name them clearly
5. Test on:
   - desktop Chrome
   - a phone sized viewport
   - GitHub Pages build (HTTPS)

## Suggested PR structure

- What changed
- Why it changed
- How to test

## Vocabulary image rules

Vocabulary images must:
- be clear and simple
- be culturally inclusive where relevant
- not contain any written words
- not contain letters or labels that give away the answer
